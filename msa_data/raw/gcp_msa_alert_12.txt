Subject: [Mandatory Service Announcement] Action Required: Compute Engine N1 Machine Type Retirement in specific regions
From: The Google Cloud Team
Date: March 09, 2026

You are receiving this notification because you are a user of Google Compute Engine N1 series virtual machines.

WHAT YOU NEED TO KNOW
As part of our continuous effort to modernize our infrastructure, Google Cloud is retiring the first-generation N1 machine types in the `us-central1-a` and `europe-west1-b` zones. 

After April 14, 2026, you will no longer be able to create new N1 instances in these specific zones. Between April 14, 2026 and April 22, 2026, any remaining N1 instances in these zones will be stopped and their host hardware will be decommissioned. 

Potential impact:
* If you have active workloads on N1 instances in the affected zones, they will experience downtime and subsequent termination if not migrated before the deadline.
* Attached persistent disks will not be deleted, but they will be detached when the host VM is retired.

WHAT YOU NEED TO DO
Action advised before April 14, 2026:
1. Identify affected resources using the Cloud Console or the included script in our migration documentation.
2. Migrate your workloads to modern machine series (N2, N2D, E2, or C3). We highly recommend the E2 series for general-purpose workloads currently on N1.
3. Update any Infrastructure as Code (Terraform, Deployment Manager) templates that explicitly reference N1 machine types in the affected zones.
